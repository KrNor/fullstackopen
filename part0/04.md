0.4: New note diagram
```mermaid
sequenceDiagram
  participant browser
  participant server

  
  browser->>server: POST request https://studies.cs.helsinki.fi/exampleapp/new_note with form data "note: "something"
  activate server
  Note left of server: the server recieves the request in path "/new_note" and checks if it is valid, and if so, saves the note along with the current time
  server-->>browser: HTML document and status code 302, and the requested resource location, in the "location" header that is "/exampleapp/notes"
  deactivate server

  browser->>server: GET request https://studies.cs.helsinki.fi/exampleapp/notes with the location in the previous response
  activate server
  server-->>browser: HTML document with status code 200
  deactivate server

  browser->>server: GET request https://studies.cs.helsinki.fi/exampleapp/notes/main.css
  activate server
  server-->>browser: css file with status code 200
  deactivate server

  browser->>server: GET request https://studies.cs.helsinki.fi/exampleapp/notes/main.js
  activate server
  server-->>browser: JavaScript file alongside status code 200
  deactivate server

  Note right of browser: browser renders the current HTML document alongside the given css
  Note right of browser: executes the JS code that initialises a new request and then sends the request 

  browser->>server: GET request https://studies.cs.helsinki.fi/exampleapp/exampleapp/data.json
  activate server
  server-->>browser: JSON file with current notes
  activate server

  Note right of browser: the browser recieves the JSON file and "onreadystatechange" function is triggered, that (if the response is valid) renders the note information
```
